Sumi Privacy Policy

What data is collected, how it is used, and who receives it

Last Updated: 2026-05-24

Definitions
Throughout this policy:
  • "Sumi", "the bot", "we", and "us" refer to the Sumi Discord bot and its operator.
  • "You" and "your" refer to the Discord user interacting with the bot.
  • "Server administrators" refers to Discord server staff with permission to manage the bot.
  • "Third-party services" refers to external APIs and services used by the bot to process requests.

1. Overview

This Privacy Policy explains what information Sumi collects, why it is collected, how it is stored, and who it may be shared with for officially operated instances.

By using Sumi, you agree to the collection and use of data as described in this policy.

Project Scope
Sumi is based on the public open-source project Freesona. This Privacy Policy only applies to bot instances operated directly by the project's maintainer and operator.

Because Freesona is publicly available, third parties may host their own independent instances with different configurations, logging practices, memory systems, hosting providers, or privacy policies.

If you are interacting with a Freesona-based bot not operated by the official maintainer, this policy may not apply to that instance.

2. Information Collected

Automatically Processed Information

Sumi processes certain Discord data in order to function properly:

  • Discord user IDs
    Used to associate memory, preferences, and interactions with the correct user.
  • Discord server (guild) IDs
    Used to separate settings, memory, and configuration between servers.
  • Message content
    Messages sent to the bot may be processed to generate AI responses and maintain conversation context.
  • Display names
    Used to identify participants in multi-user conversations.

Optionally Stored Information

  • Long-term memory facts
    If long-term memory is enabled, Sumi may extract information from conversations such as preferences, interests, or recurring details to personalise future responses.
  • RSS feed configuration
    Feed names and URLs added by server administrators are stored for server functionality.

3. How Information Is Used

Sumi uses collected information to:

  • Generate context-aware AI responses
  • Maintain temporary conversation memory during active sessions
  • Personalise future responses if long-term memory is enabled
  • Store server-specific settings and configuration
  • Operate features such as RSS feeds, automation, and moderation-related functions

4. Data Storage and Retention

Short-Term Memory

Temporary conversation memory is stored in active bot memory while the bot is running. This memory may be cleared:

  • When the bot restarts
  • When a server administrator uses /clearmemory
  • When conversation context expires internally

Long-Term Memory

Long-term memory data is stored locally in memory.json on the bot's host server. This data remains stored until manually removed by the bot operator.

Configuration Data

Server configuration is stored in config.json and persists across restarts unless manually deleted.

5. Third-Party Services

Some requests require sending data to external services in order for features to work correctly.

Depending on the command or interaction, your message content or uploaded files may be transmitted to:

  • Google Gemini
    Used for AI response generation and optional memory extraction.
  • Wolfram Alpha
    Used for mathematics and computational queries.
  • Google Custom Search
    Used for web search functionality.
  • MVSEP
    Used for audio separation requests involving uploaded audio files.

These third-party services operate under their own privacy policies and data handling practices. Sumi does not control how external providers process transmitted data.

Sumi does not sell user data.

6. Data Security

Data is stored on the bot hosting provider, such as Railway or an equivalent hosting platform.

Reasonable precautions are taken to protect stored data, but no online system can guarantee complete security. Use of the bot is at your own risk.

7. Your Rights and Data Removal

You may request removal of stored data at any time.

  • Server administrators may clear temporary session memory using /clearmemory
  • Long-term memory removal requires manual action by the bot operator
  • You may contact the bot operator through the server's support channels for data requests

8. Children's Privacy

Sumi is not intended for use by individuals under the age of 13, or below the minimum age required by applicable law in their jurisdiction.

If you believe data belonging to a minor has been collected, contact the bot operator immediately.

9. Changes to This Policy

This Privacy Policy may be updated at any time without prior notice.

Continued use of Sumi after changes are published constitutes acceptance of the revised policy.

10. Contact

For questions, concerns, or data-related requests, contact the bot operator through the designated support channels provided in the Discord server.

By using Sumi, you acknowledge that you have read and understood this Privacy Policy.
